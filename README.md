# workkflow_boilerplate

## Purpose

This repository is a documentation-first boilerplate for capturing business context, marketing guidance, operating workflows, reference material, and agent instructions.

It currently contains documentation and local agent configuration only. There is no application framework, application source code, runtime entry point, route layer, model layer, controller layer, view layer, test suite, or build/deployment script in the repository root.

## Structure

- `AGENT.md` gives high-level guidance for agents working in this repository.
- `docs/` is the business and workflow source of truth.
- `.opencode/` contains local opencode agent configuration and dependencies. Treat this as tooling/configuration, not business documentation.

## How To Use

Start with `AGENT.md`, then read `docs/README.md` and the relevant section README files before making business, marketing, operations, or workflow decisions.
