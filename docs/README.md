# Business Documentation

## Purpose

This folder is the business source of truth for humans and AI agents.

## What Belongs Here

Use this folder for stable business context, operating workflows, marketing guidance, reference material, and decisions that should guide work across codebases.

## Structure

- `business/` contains the stable business source of truth: overview, focus, audience, positioning, product strategy, pricing, and principles.
- `marketing/` contains marketing strategy, brand voice, content workflow, blog, social, newsletter, and magnet guidance. Blog post work should live under `marketing/blog/`; draft or published post files should live under `marketing/blog/posts/`.
- `operations/` contains repeatable workflows and recurring operational tasks.
- `reference/` contains shared definitions and recorded decisions.

## Repository Context

This repository is currently a documentation boilerplate. It does not include application source code, an application framework, routes, models, controllers, views, tests, scripts, or deployment configuration. If implementation files are added later, update this documentation so the repository structure and operating assumptions remain accurate.

## How To Use This Section

Read this folder before making assumptions about the business, its audience, its positioning, its marketing, or its workflows. Start with `business/`, then move to the relevant workflow or marketing section.
